# 发布前检查清单

## 自动检查（必须过）

```bash
# 1. 内容质量门
node scripts/pre-publish-check.js blog/your-article.html

# 2. 中文检查（EN 页面）
grep -n '[一-龥]' blog/your-article.html | grep -v 'zh:' | grep -v '中文'
# 应该无输出
```

## 手动检查（快速过一遍）

- [ ] 标题是否准确概括正文，不是标题党
- [ ] 所有价格、版本号、链接是否可点击/可验证
- [ ] 是否有至少一个 "读者可以立即做的动作"
- [ ] 最后更新时间是否已标注
- [ ] 是否有缺点/限制（不能只夸）
- [ ] 对比表格是否 ≥5 个维度（评测类）

## 不通过 = 不发布

| 分数 | 结果 |
|------|------|
| ≥80% | ✅ 发布 |
| 60-79% | ⚠️ 修完警告再发布 |
| <60% | ❌ 重写 |
